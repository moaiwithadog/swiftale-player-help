## 1. SWIFTALE PLAYER と LLM について

### BYOL（Bring Your Own LLM）方式

SWIFTALE PLAYER は、**プレイヤー自身が用意した LLM を GM（ゲームマスター）役として使う** BYOL 方式を採用しています。
このアプリの開発者サーバーに入力内容が送られることはありません。

LLM の選択肢は2種類あります。

| | Ollama（ローカル） | クラウド API |
|---|---|---|
| 動作場所 | このPC上（初期設定） | 各社のクラウドサーバー |
| 外部への送信 | 設定次第（初期値はこのPC） | ゲーム内容が各プロバイダへ送信される |
| 課金 | Ollama 自体の費用は発生しない | API 利用料が発生し得る |
| 事前準備 | Ollama のインストール＋モデル取得 | アカウント作成＋ API キー発行 |
| 向いているケース | 外部送信・課金を避けたい、PC に余裕がある | 設定をすぐ済ませたい、高品質な出力が欲しい |

### 迷ったときの選び方

- **インストール作業を避けたい、とにかく早く試したい** → クラウド API（各プロバイダで API キー取得と利用料の発生を伴います）
- **外部送信や API 課金を避けたい** → Ollama
- **まだどちらか分からない** → アプリを起動して「今は設定せず戻る」を選び、後から設定できます

> **注意**: Ollama を選んでも、「外部送信なし・無料・準備不要」ではありません。  
> Ollama 本体とモデルのインストール、PC の空き容量（数 GB 以上）と処理性能が必要です。

## 2. Ollama を使う（ローカル）

### 2-1. Ollama のインストール

[Ollama 公式サイト](https://ollama.com/) からインストーラをダウンロードして実行します。

**Windows**
1. 公式サイトから Windows 用インストーラ（`.exe`）をダウンロードする
2. インストーラを実行し、画面の指示に従う
3. インストール後、タスクトレイに Ollama のアイコンが表示されれば起動中

**macOS**
1. 公式サイトから macOS 用インストーラ（`.dmg`）をダウンロードする
2. `.dmg` を開き、Ollama をアプリケーションフォルダへドラッグする
3. 初回起動時にメニューバーにアイコンが表示されれば起動中

**Linux**
```bash
curl -fsSL https://ollama.com/install.sh | sh
```
インストール後、`ollama serve` でサービスを起動するか、systemd 経由で自動起動する。

---

### 2-2. モデルのインストール

Ollama を起動したら、使用するモデルをダウンロードします。

#### 推奨モデル

| モデル | サイズ目安 | 用途 |
|---|---|---|
| `gemma4:e2b` | 約 2GB | **初期候補。** 標準的な PC でも動作しやすい |
| `gemma4:e4b` | 約 4GB | PC に余裕がある場合の候補 |
| `gemma4` | 約 8GB 以上 | より高い出力品質を求める場合 |

> サイズはモデルのバージョンにより変わります。Ollama 公式サイトで最新情報を確認してください。

#### インストール方法

**Ollama のアプリ画面から**（推奨）
Ollama のアプリ画面でモデル名を検索してインストールできます。手順はインストーラのバージョンによって異なります。

**コマンドラインから**（上級者向け）
```bash
ollama run gemma4:e2b
```
初回実行時にモデルが自動ダウンロードされます。インストール済みのモデル一覧は `ollama list` で確認できます。

---

### 2-3. アプリでの設定

1. SWIFTALE PLAYER を起動する
2. タイトル画面で「START」を押す
3. セットアップウィザードが開いたら「ローカルで使う（Ollama）」を選ぶ
4. Ollama URL は通常そのまま（`http://127.0.0.1:11434`）でよい
5. 「モデル一覧を確認」ボタンを押してモデルが表示されることを確認する
6. 使用するモデルを選んで「設定を保存」する

> セットアップ完了後、「ワールド一覧へ」から同梱のサンプルを選んで遊べます。

---

### 2-4. Ollama の URL について

Ollama の URL は、接続先の Ollama サーバーのアドレスです。

- **初期値 `http://127.0.0.1:11434`** は、このPC上の Ollama に接続する設定です。通常はこのままで使ってください。
- URL を `localhost` / `127.0.0.1` 以外に変更すると、ゲーム中の入力やセッション文脈が変更先のサーバーへ送信されます。
- よく分からない場合は変更しないでください。

---

### 2-5. Ollama のよくある問題

**「Ollama に接続できません」と表示される**
- Ollama アプリが起動しているか確認してください（Windows ならタスクトレイ、macOS ならメニューバーにアイコンがあるか）
- 起動していない場合は Ollama を起動してから「モデル一覧を確認」を再試行してください

**「モデルが見つかりません」と表示される**
- コマンドラインで `ollama list` を実行し、インストール済みのモデル一覧を確認してください
- リストにないモデルは `ollama run <モデル名>` でインストールしてください

**モデルのインストールに時間がかかる**
- モデルのダウンロードは数 GB になる場合があります。完了するまでお待ちください
- ダウンロードが途中で止まった場合は、コマンドを再実行すると再開できます

**「URL が localhost ではありません」と警告が出る**
- 接続先がこのPC以外の Ollama の場合、ゲーム内容がそのサーバーへ送信されます
- 信頼できるサーバーかどうかを確認したうえで設定してください

## 3. クラウド API を使う

### 3-0. API キーとは

**API キー**は、各社の AI サービスをこのアプリから使うための秘密の認証文字列です。

- ChatGPT Plus、Claude Pro、Gemini Advanced などの**月額プランのログイン情報（メールアドレス・パスワード）とは別のものです**
- API キーは各社の開発者向けダッシュボードで別途発行します
- API キーがあれば、月額プランに加入していなくても、使った分だけ API を利用できます（従量課金）

**API 利用料について**
- 一般にテキスト生成 API は入出力テキスト量に応じて課金されます
- 各社の無料枠や料金体系は変わることがあるため、必ず各プロバイダの最新情報を確認してください
- 料金が不安な場合は **Ollama** を使ってください（API 利用料は発生しません）

**API キーの保管**
- このアプリは API キーを OS の資格情報ストア（Windows Credential Manager など）に保存します
- 設定画面に API キーの値は再表示されません
- このアプリ専用の API キーを作成することを強くおすすめします
- プロバイダ側で利用上限（月額上限額など）を設定しておくことを推奨します

---

### 3-1. Google Gemini

**API キーの取得手順**
1. [Google AI Studio](https://aistudio.google.com/) にアクセスする
2. Google アカウントでログインする（Gmail アカウントがあれば作成不要）
3. 「Get API key」または「APIキーを取得」ボタンを押す
4. 新しいプロジェクトを作成するか、既存のプロジェクトを選んで API キーを発行する
5. 表示された API キー（`AIza...` から始まる文字列）をコピーする

**アプリでの設定**
1. ウィザードで「API で使う」を選ぶ
2. プロバイダ選択で「Google Gemini」を選ぶ
3. 発行した API キーを入力して保存する

**推奨モデル**（目安。最新情報は Google AI Studio で確認してください）
- `gemini-2.5-flash` — 高速・軽量
- `gemini-2.5-pro` — より高い品質を求める場合

> **料金・利用上限**: プランごとのレート上限・料金体系は [Google AI Studio の料金ページ](https://ai.google.dev/pricing) で最新情報を確認してください。

---

### 3-2. Anthropic Claude API

**API キーの取得手順**
1. [Anthropic Console](https://console.anthropic.com/) にアクセスする
2. アカウントを作成してログインする
3. 「API Keys」メニューから新しい API キーを発行する
4. 表示された API キー（`sk-ant-...` から始まる文字列）をコピーする（再表示されないので必ず控える）

**推奨モデル**（目安）
- `claude-haiku-4-5-20251001` — 高速・低コスト
- `claude-sonnet-4-5` — バランス型

---

### 3-3. OpenAI（現時点では非推奨）

> **ご注意**: OpenAI のモデルは、本アプリの構造化出力やシナリオ生成の制約への追従が不安定になりやすく、序盤で物語の謎が崩れる・スコープ制約を外れるなどの挙動が確認されています。**現時点では非推奨**です。クラウド API を使う場合は Gemini / Claude / Grok を推奨します。設定自体は可能なので、以下に手順を残します。

**API キーの取得手順**
1. [OpenAI Platform](https://platform.openai.com/) にアクセスする
2. アカウントを作成してログインする
3. 「API keys」メニューから新しい API キーを発行する
4. 表示された API キー（`sk-...` から始まる文字列）をコピーする（再表示されないので必ず控える）

**推奨モデル**（目安）
- `gpt-4o-mini` — 高速・低コスト
- `gpt-4o` — 高品質

---

### 3-4. xAI Grok

**API キーの取得手順**
1. [xAI Console](https://console.x.ai/) にアクセスする
2. アカウントを作成してログインする
3. API キーを発行する（`xai-...` から始まる文字列）

**推奨モデル**（目安）

---

### 3-5. API キーの形式について

各プロバイダの API キーには典型的な文字列の形式があります。

| プロバイダ | 典型的な形式 |
|---|---|
| Google Gemini | `AIza...` |
| Anthropic Claude | `sk-ant-...` |
| OpenAI | `sk-...` |
| xAI Grok | `xai-...` |

入力した文字列がこの形式と異なる場合、ウィザードで警告が表示されます。  
ChatGPT・Claude・Gemini などの**月額プランのパスワードや、メールアドレスではありません。**  
API キーがまだ分からない場合は「今は設定せず戻る」で一旦戻り、後から設定を再開できます。

## 4. プライバシーとデータの扱い

### このアプリが送信するデータ

| 利用方式 | 送信先 | 送信される内容 |
|---|---|---|
| Ollama（初期設定） | このPC上の Ollama のみ | プレイヤー入力、セッション文脈 |
| Ollama（URL変更時） | 指定した Ollama サーバー | プレイヤー入力、セッション文脈 |
| クラウド API | 選択したプロバイダのサーバー | プレイヤー入力、セッション文脈 |

### 開発者サーバーへの送信はありません

このアプリは BYOL（Bring Your Own LLM）方式です。  
**ゲームデータ・プレイ内容・API キーが SWIFTALE PLAYER の開発者サーバーへ送信されることはありません。**

### ゲームデータの保存場所

ワールド・キャンペーン・セッションのデータは、このPC上のローカルデータベース（SQLite）に保存されます。

### AI 生成コンテンツへのガードレール

このアプリには、不適切なコンテンツの生成を抑制するための安全ゲートが組み込まれています。  
送信前のプレイヤー入力と、表示・保存前の AI 出力の両方を検査・サニタイズします。

## 5. よくある質問（FAQ）

**Q. 完全無料で使えますか？**  
A. Ollama を使う場合、このアプリから API 利用料は発生しません。ただし Ollama 本体・モデルのインストール、PC の性能と空き容量が必要です。クラウド API は各プロバイダで利用料が発生します。

**Q. API キーを入力するとすぐ課金されますか？**  
A. API キーを保存しただけでは課金されません。セッション中に AI が呼び出されたときに、使った分だけ課金されます。プロバイダ側で利用上限を設定しておくと安心です。

**Q. どのモデルを選べばいいか分かりません**  
A. 迷った場合は、Ollama なら `gemma4:e2b`、クラウドなら Gemini の `gemini-2.5-flash` から始めてみてください。設定はあとからいつでも変えられます。

**Q. Ollama と API は後から切り替えられますか？**  
A. はい。Settings 画面または セットアップウィザード（START → セットアップ）からいつでも変更できます。

**Q. API キーがどこで作れるか分かりません**  
A. 各プロバイダの手順は [3. クラウド API を使う](#3-クラウド-api-を使う) にまとめています。API キーはアプリ内では作れず、各プロバイダのウェブサイトで作業が必要です。

**Q. ゲームデータはどこに保存されますか？**  
A. このPC上のローカルデータベースに保存されます。クラウドへの自動バックアップはありません。

**Q. 別の LLM（上記以外）は使えますか？**  
A. 現在サポートしているのは Ollama・Claude・OpenAI・Gemini・Grok です。OpenAI 互換の API エンドポイントを持つサービスは、設定によって動作する場合がありますが、動作保証はありません。
