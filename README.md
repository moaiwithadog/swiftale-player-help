## これは何か

SWIFTALE PLAYER は、生成AIにゲームマスター(GM)役を務めてもらって、ひとりで遊ぶことができる、テキスト TRPG アプリです。

ルールブックは、生成AIをGMにするため専用のオリジナル品を使っています。
 -> [生成AIにTRPGやらせてみた [SWIFTALE]](https://ncode.syosetu.com/n4791lq/)

世界観に制限はありません。
剣と魔法のナーロッパでも、東京に生きる妖怪ハンターでも、恒星間運送業者でも、お江戸の隠密同心でも、なんでもかんでも、お好みのままです。

行動宣言は、AIがいくつか例を挙げてくれるので、それをピックアップするだけでよいです。選択肢が自動生成されるゲームブックみたいな風情ですね。

もちろん、自由に行動宣言してもOK。台詞のロールプレイやPCの心理描写などと合わせて、AIに投げ返してあげれば、AIのGMが拾ってくれて、判定や伏線に活かしてくれたりもします。

同じ世界観で、連続するシナリオを、キャンペーンとして長期間、遊ぶことも可能です。

### BYO-AI（Bring Your Own AI）方式

このアプリは、プレイヤーが用意した 生成AI(LLM) を 使う ** BYO-AI ** 方式を採用しています。

- 自分の PC 上で動かす **Ollama（ローカル）**
- 各社のクラウドサービスを使う **クラウド API**（Claude / OpenAI / Gemini / Grok）

どちらを選ぶかは、お好みです。
**なお、このアプリの開発者サーバーにゲームデータや入力内容が送られることはありません。**

### ゲーム中の物語構造

物語は3つの階層で構成されます。

```
🌎ワールド（世界観）
  └─ 📖キャンペーン（ひとつながりの大きな物語）
       └─ 📜セッション（物語の一章 / クエスト1回分に相当）
```

📜セッションは1回あたり30分程度を想定。サクッと遊べます。

📖キャンペーン内の📜セッションは、ひとつの時系列でつながり、前回の結果を引き継いで次の冒険へ進めます。
だいたい5-6セッションで1キャンペーン、といったつもりで設計してます。

プレイ中に確定したことは「固定ルール」として、その🌎ワールドに積み重なっていきます。
📜セッションや📖キャンペーンを重ねて、きみだけの🌎ワールドを創っちゃいましょう。

リプレイ出力機能もあるので、いいかんじのものができたら、どこかに公開したり、自由にしてもらって構いません。
それは、あなたが紡いだ物語なのですから。

### ルールと画面の説明

プレイヤーは、主人公(プレイヤー・キャラクター / PC )1人だけを操作します。
PCは、📖キャンペーンの単位で設定できます。

PCのステータスは、3つだけ。
- HP: ヒット・ポイント。無くなると行動不能。
- SP: スペシャル・ポイント。気力とか魔力とか、そういうものです。0でも行動はできるが、弱い。
- Tension: 危機/恐怖/焦り/汚染など。上がるほど不利、10を超えると、何らかの事故が起きたのち半減します。
(名作「クトゥルフの呼び声」のSAN値に近いものです、「何か破滅的なことが迫っている」というのを数値化することで、プレイヤーとGMで緊張感を共有し、AIのGMに展開を作らせやすくしてあります。)

行動は選択式ですが、日本語で入力もできます。
お好みのロールプレイでGMを篭絡して、うまいことゲームを進めてください。

小休息するとHP・SPが+1回復。連続して小休憩はできますが、回復は増えません。
大休息するとHP・SP全回復、Tension0に戻ります。安全な場所で十分に休める場合だけ可能です。

1回の行動で1Turnが過ぎます。いちおう目安のTurn数がありますが、無視してゲーム継続もできます。
🔑マークは、何かキーアイテムかキーパーソンかに出会った印です。
🚩マークは、何かゴールに向けて一歩進んだ印です。

ゴールに辿り着いた/ゴールに辿り着けないことが明らか/推奨Turnを大幅に超えると、GMが「もう終わりにする？」と聞いてきます。
プレイヤーはGMを無視して📜セッションを続行することもできるし、いったん終わらせて次の📜セッションに引き継がせることもできます。
物語の区切りが付いたら、📖キャンペーンを終了させることができます。GMがエンディングを語ってくれます。

| | 要件 |

## はじめかた

### Step 1. アプリをインストールして起動する

ダウンロードしたインストーラを実行し、アプリを起動します。
タイトル画面が表示されたら起動成功です。

### Step 2. 生成AI(LLM) を設定する

「START」を押すと、**初回セットアップウィザード** が開きます。

ウィザードでは、LLM の利用方式（Ollama またはクラウド API）を選んで設定します。

**迷ったときは**：
- インストール作業を避けたい → **Gemini（クラウド API）** の無料枠が最短導線
- 外部送信・API 課金を避けたい → **Ollama**（ローカル）

設定の詳細手順は **[LLM セットアップガイド](./llm-setup.md)** にまとめています。

### Step 3. サンプルセッションを遊んでみる

セットアップ完了後、「**サンプルセッションを試す**」ボタンで同梱のサンプルシナリオをすぐ遊べます。
世界観の理解やアプリの操作感をつかむための導入シナリオです。

## 生成AI(LLM) の設定

このアプリで GM として使える LLM は2種類あります。

### Ollama（ローカル）

自分の PC 上で LLM を動かします。
ゲーム内容が外部サービスへ送信されず、API 利用料も発生しません（Ollama 自体は無料）。
ただし Ollama 本体とモデルのインストール、PC の空き容量・処理性能が必要です。

### クラウド API

各社の API を使います。
インストール作業は不要で、高品質な出力が期待できます。
ゲーム中の入力とセッション内容が選択したプロバイダのサーバーへ送信され、API 利用料が発生し得ます。

| プロバイダ | 特記事項 |
|---|---|
| Anthropic Claude | 表現力が抜群。利用料が少し高いかな。 |
| OpenAI | 生成AIの元祖。ちょっぴり真面目すぎるかも。 |
| Google Gemini | Googleの誇る対応力。ちょっと適当なところもあるみたい。 |
| xAI Grok | X(旧Twitter)由来の知識量。意外と律儀かも。 |

各プロバイダの API キー取得手順・推奨モデル・よくある問題は  
**[→ LLM セットアップガイド](./llm-setup.md)** で詳しく説明しています。

## このゲームが使う AI について

### データの送信先

| 利用方式 | 送信先 |
|---|---|
| Ollama（初期設定） | このPC上の Ollama のみ |
| クラウド API | 選択したプロバイダのサーバー（Google / Anthropic / OpenAI / xAI） |

このアプリは BYO-AI 方式のため、**SWIFTALE PLAYER の開発者へゲームデータが送られることはありません**。
クラウド API を選んだ場合、プレイヤーの入力内容とセッションの文脈は、選択したプロバイダへ送信されます。
各プロバイダのプライバシーポリシーも併せて確認してください。

### ガードレール（安全ゲート）

このアプリには、不適切なコンテンツの生成を抑制するための安全ゲートが組み込まれています。

- プレイヤーの入力内容は **送信前** に検査されます
- AI の出力内容は **表示・保存前** に検査・サニタイズされます
- AI を使わないルールベースの判定で動作します

初期方針は「日本の一般的な少年向け冒険作品程度」を上限とする SFW/General です。

### AI 生成コンテンツの性質

このアプリはプレイ中にテキストをリアルタイム生成します（Live-Generated AI Content）。
生成される描写・裁定・行動候補は AI によるものであり、開発者がすべての出力を事前に確認・保証するものではありません。

## よくある質問

**Q. 完全無料で使えますか？**  
A. Ollama を使う場合、API 利用料は発生しません。ただし Ollama とモデルのインストール、PC の性能・空き容量が必要です。クラウド API では、一部のプロバイダでは無料枠を利用できる場合がありますが、各プロバイダの情報を確認ください。

**Q. API キーを持っていません。クラウド API は使えますか？**  
A. API キーは各社のウェブサイトで別途発行が必要です。ChatGPT Plus などの月額プランのログイン情報とは別のものです。詳しくは [LLM セットアップガイド §3](./llm-setup.md#3-クラウド-api-を使う) を参照してください。

**Q. Ollama とクラウド API は後から切り替えられますか？**  
A. はい。Settings 画面またはセットアップウィザードからいつでも変更できます。

**Q. PC のスペックが低いのですが動きますか？**  
A. クラウド API を使うなら、だいたい動きます。(AI 処理がネット上のサーバー側で行われるため PC スペックに依存しません。) Ollama を使う場合、ゲーミングPCじゃないと辛いと思います。

**Q. ゲームデータはどこに保存されますか？**  
A. 実行したPC上のローカルデータベースに保存されます。クラウドへの自動バックアップ機能はありません。

**Q. 日本語以外の言語で遊べますか？**  
A. 現在は日本語を主言語として開発しています。開発者は日本語話者であり、日本語以外で安全ゲートを確保できないのが、その理由です。

その他の質問は [LLM セットアップガイド §5（FAQ）](./llm-setup.md#5-よくある質問faq) も参照してください。
